Eclipse defined
C Function clInit

Syntax:

clInit initial_stack_size => void

Arguments and Values:

initial_stack_size---an unsigned int.

Description:

Initializes all data used by, and available from, the Eclipse run time library (2), and prepares the system for use.

Space for initial_stack_size entries are allocated for the Lisp control stack (i.e. that which keeps track of bindings, catch tags, unwind-protects, etc.) If 0 is specified for initial_stack_size, an implementation-specific default value is used.

Side Effects:

After clInit returns, some Lisp external and internal symbols used by the the run time library are interned, bound and fbound appropriately, and have correct property lists. However, clInit is not required to reinitialize any such data if called repeatedly, or after the system has already been initialized in some other way.

Affected By: None.

Exceptional Situations: None.

See Also:

Initializing the Eclipse Library, clInitD, Lisp Top-Level.

Notes:

The default initial_stack_size is currently 1024.

Any stand-alone application (1), (both dynamic applications and run time applications,) must be initialized by calling clInit.
