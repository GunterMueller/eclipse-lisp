Description:

The following classes must be used as superclasses of user-defined stream classes. They are not intended to be directly instantiated; they just provide places to hang default methods.

Fundamental-stream is the base class for all CLIM streams. Eclipse further defines it to represent an actual stream directly connected to an operating system stream or to a string-stream. It does not represent an encapsulating-stream.

Fundamental-input-stream implements input streams. Programmers may define methods for stream-clear-input.

Fundamental-output-stream implements output streams. Programmers may define methods for stream-clear-output, stream-force-output, and stream-finish-output.

Fundamental-character-stream implements character streams, including the stream-element-type accessor. Character streams can be created by defining a class that includes either fundamental-character-input-stream or fundamental-character-output-stream (or both) and arranging to have stream-element-type return a subtype of character.

Fundamental-character-input-stream implements character input streams. Programmers must define methods for stream-read-char. Programmers may also define methods for stream-read-sequence.

Fundamental-character-output-stream implements character output streams. Programmers must define methods for stream-write-char and stream-line-column. Programmers may also define methods for stream-start-line-p, stream-advance-to-column, stream-terpri, stream-fresh-line, stream-write-sequence, and stream-write-string.

Fundamental-binary-stream implements binary streams. Binary streams can be created by defining a class that includes either fundamental-binary-input-stream or fundamental-binary-output-stream (or both) and arranging to have stream-element-type return a subtype of integer.

Fundamental-binary-input-stream implements binary input streams. Programmers must define methods for stream-read-byte. Programmers may also define methods for stream-read-sequence.

Fundamental-binary-output-stream implements binary output streams. Programmers must define methods for stream-write-byte. Programmers may also define methods for stream-write-sequence.

Notes:

The basic stream-based input and output protocol used by CLIM is taken from the "STREAM-DEFINITION-BY-USER" proposal to the X3J13 committee, made by David Gray of Texas Instruments. This proposal was not accepted by the X3J13 committee as part of the ANSI Common Lisp language definition, but many Lisp implementations do support it.

The CLIM specification does not define the precise stream class-precedence-list of these classes; it only mentions some required superclass. The Eclipse-defined class-precedence-list specified here is consistent with the requirements of CLIM.
