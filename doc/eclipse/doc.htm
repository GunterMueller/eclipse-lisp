Eclipse Introduction: Organization of the Documentation

The "Introductory Material" includes:
  • This and other information about the documentation itself, and about Eclipse in general.
  • References to information on Common Lisp, including standard references. The home page of the Association of Lisp Users (ALU) contains such references and other information. A mirror of the ALU web-site is included with this distribution of Eclipse.
  • License, copyright and conformance statements.
  • Installation instructions.
  • Release notes about the current version.

A glossary of terms used by the Eclipse documentation which do not appear in the standard are included in the "Eclipse-Specific Glossary".

The remaining Eclipse documentation is divided into three groups:

  1. Documentation of Eclipse-specific aspects of Common Lisp. This is of interest to all users of Eclipse, and includes:
    • Aspects of Common Lisp which the standard specifies as being defined by the implementation.
    • Documentation of permitted extensions to Common Lisp.
    • Behavior of the Lisp top-level provided by the eclipse (1) program, such as the reading of initialization files.
    • A dictionary of Eclipse-defined functions, variables and other Lisp utilities.

  2. C-specific information, such as:
    • The representation of Lisp data as C data.
    • The argument passing convention used by functions in the library.
    • The naming convention by which Lisp symbols are converted to C identifiers.
    • Initializing Lisp data in C applications.
    This section also includes some brief material for C programmers who are not familiar with Common Lisp.

  3. The creation of separate, stand-alone applications which do not need the eclipse (1) program to run. This combines elements from both the other sections.
    • It might be used by Lisp programmers in creating a new, customized Lisp top-level, perhaps incorporating some user defined C functions.
    • It might be used by C programmers in incorporating their C or Lisp code into some other, largely C, application.
    This includes the additional options to compile-file that support the generation of C source files from Lisp source.
This organization follows the product organization described in "Organization of Eclipse".
