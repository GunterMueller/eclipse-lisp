What's New for Eclipse Common Lisp Release 1.0.2

December 5, 1997

See Also:
  • 1.0.1 (Solaris 2, Unix System V)
  • 1.0 (SunOS 4.1, Berkeley Unix)

Features

  • Ported to HP PA-RISC/HP-UX. (First non-Sun Eclipse platform.)
  • Separate binary directories for each platform, with links to master C source directory.
  • A single macro in eclipse.h defines the Eclipse version as a C string.
  • Version 1.0.1 licence holders can upgrade to 1.0.2 by downloading ftp://www.elwood.com/eclipse/patch-1.0.1.tgz to the general (i.e. non-versioned) directory in which eclipse is installed, and following the directions in "UNPACKING" for downloaded file distributions. The libraries and executable can then be regenerated following the directions for recompilation in the release notes.

Bug fixes

  • Reading (or loading) files with :external-format :multi-byte (which is the default) now works with files containg more than 512 bytes.
  • Read can now properly handle a list containing a float which begins with a decimal (eg. .5 as opposed to 0.5).
  • 29 bit fixnums now function properly.
  • Truncate and ftruncate rounds in the correct direction.
  • Compiled-function-p works correctly.
  • Load-time-value in interpreted code now returns a single value, just as in compiled code.
  • Local variables named with uninterned symbols are prefixed with "L" when compiled to C, not "CL".
  • Formatting error in time macro is fixed.
  • A patch is available that provides the following additional bug fixes:
    • Apply checks that call-arguments-limit is not exceeded. (Note the special pretty printer bug, in the new features for release 1.0.3.)
    • The scanning of directories no longer involves the Posix opendir() utility, which can interfere with the garbage collector.
    • Dribble is now correct.
    • Trace now correctly prints the traced function name.
    • The variable eclipse::not-found is now proclaimed special. This variable appears within macro generated code for with-package-iterator, do-symbols, etc, where it is assumed to be special.
    • The first use of time no longer reports a negative real time, and the value reported on the HP is no longer off by a constant factor.
    • Uninterned symbols appearing as literals in a file being compiled now generate references to properly initialized, static clObjects, instead of references to undefined extern clObjects. Uninterned symbols appear as literals in some macroexpansions, such as the expansion of a defstruct which uses the :print-function option.
    • Echo streams don't try to echo NIL to the output stream when read-char-no-hang is called and no character is available.
    Eclipse 1.0.2 users can upgrade to 1.0.2A by downloading ftp://www.elwood.com/eclipse/patch-1.0.2.tgz to the general (i.e. non-versioned) directory in which eclipse is installed, and following the directions in "UNPACKING" for downloaded file distributions. The libraries and executable can then be regenerated following the directions for recompilation, in the release notes.
