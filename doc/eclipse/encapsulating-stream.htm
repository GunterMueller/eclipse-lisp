CLIM defined
Class ENCAPSULATING-STREAM,

Class Precedence List:

encapsulating-stream, stream, standard-object, t

Description:

An encapsulating stream is a special kind of stream that "closes over" another stream, handling some of the usual stream protocol operations itself, and delegating the remaining operations to the "encapsulated" stream. Encapsulating streams may be used by some CLIM implementations in order to facilitate the implementation of features that require the dynamic modification of a stream's state and operations.

Encapsulating-stream classes must provide "trampoline" methods for all stream protocol operations in which they will participate. These "trampolines" will simply call the same generic function on the encapsulated stream.

Initialization:

:stream All encapsulating streams must handle the :stream initarg, which used to specify the stream to be encapsulated.

Readers:

encapsulating-stream-stream stream
Primary Method encapsulating-stream-stream (stream encapsulating-stream)

Returns the stream encapsulated by the encapsulating-stream stream.

See Also:

"Extensions to Common Lisp", CLIM stream classes, Eclipse stream classes, stream-element-type, open-stream-p close.

Notes:

CLIM implementations need not use encapsulating streams at all. If encapsulating streams are used, they must adhere to the the encapsulating-stream protocol. Encapsulating streams are not part of CLIM's API.

The CLIM specification does not define the precise class-precedence-list of this classes. The Eclipse-defined class-precedence-list specified here is consistent with the requirements of CLIM.

Eclipse may use encapsulating-stream to implement various ANSI composite streams.
