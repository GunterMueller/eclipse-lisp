Eclipse Lisp: The Lisp Top-Level

Eclipse has a stand-alone application called eclipse that provides the functionality of a traditional Lisp top-level.

This program takes a number of command switches or options. How a user indicates to the operating system that one or more switches should be provided depends on the platform. For example, on Unix systems the letter corresponding to each switch is provided on the command line, preceded by a dash. Eg. eclipse -s -i my-init.lisp

Operating system switches:

s Silent mode.
n No init file.
i init-file Load the specified init-file rather than performing the usual file system search for one.
v Prints the version and immediately exits with a zero status.

Description:

After parsing command switches, the eclipse program first initializes the Lisp system, as though by calling clInit and clInitD.

It then starts a Lisp top-level (2) by calling clLispTopLevel.

The verbose argument to clLispTopLevel is false if the s switch was given to the eclipse program. Otherwise it is true.

The init argument to clLispTopLevel is false if the n switch was given to the eclipse program. Otherwise, the init argument will be a string designating the init-file following any i switch, or the symbol T if no i switch is specified.

Invoking the exit restart exits clLispTopLevel with a status value supplied when invoking the restart. The eclipse program then exits, returning a non-zero status to the operating system if status was true, and zero if status was false.
