Eclipse defined
Variable *PROMPT-HOOK*

Value Type:

a function of three arguments, as defined below.

Initial Values:

A function that prints a fresh-line, the current debug level if non-zero, and the string "> " to the specified stream.

Description:

Forms to be evaluated by the read-eval-print loop are prompted for by calling the function which is the value of *prompt-hook*. The function is called with three arguments:

  • The stream to which the prompt should be printed.
  • The recursive debug level. The Lisp top-level is at level 0, the first entry to the debugger at level 1, etc.
  • The condition which caused entry to the read-eval-print loop to be entered (i.e. the condition passed to invoke-debugger).
Affected By:

Might be set by the user in init.lisp.

See Also:

clLispTopLevel, init.lisp.

Notes:

Some users prefer to have a prompt that indicates such information as the current *package*, time, machine-instance, *default-pathname-defaults*, etc. While some of this information is not usually needed by Lisp programmers, the ability to change the prompt for the standard read-eval-print loop makes it easier to use this loop to provide end-user interaction in delivered applications.
