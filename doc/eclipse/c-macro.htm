Compound Type Specifier C:MACRO

Compound Type Specifier Syntax:

c:macro arg-typespec value-typespec [filename]

Compound Type Specifier Arguments:

arg-typespec---a list of zero or more type specifiers.

value-typespec---a type-specifier.

filename--a string denoting the name of an include file.

Compound Type Specifier Description:

The list form of the c:macro type-specifier can be used only for declaration and not for discrimination. Every element of this type is a function-like macro that accepts arguments of the types specified by the argj-types and returns values that are members of the types specified by value-type.

When a call to a function that has been declared with a c:macro type specifier is translated to C using compile-file, the generated C code will use calling conventions appropriate for a call to a C function-like macro.

The filename, if present, will appear in an #include cpp directive at the head of any C file generated by compile-file that contains a call to a function declared using this compound type specifier. Any quotation marks ("filename" ) or brackets (<filename>) that should appear in the #include directive must appear within the string denoted by filename.

See Also:

The function type specifier in the Common Lisp standard, "Delivering Applications", "Lisp Function Calls in C", c:function.

Notes:

Unlike the ordinary Lisp function compound type specifier, the &optional, &rest, &key, and &allow-other-keys markers cannot appear in the list of argument types.

It may be useful to use symbols in the C package to identity type specifiers for use in this compound type specifier.
