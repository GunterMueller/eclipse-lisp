Restart EXIT

Data Arguments Accepted:

status--a generalized boolean. The default is NIL.

exitp--a generalized boolean. The default is T.

Description:

The exit restart is used by a top-level application loop (such as the Lisp top-level) to provide a way to cleanly exit the application.

If exitp is true, the status argument is returned as the value of the application that defined the restart.

If exitp is false, NIL is returned to the caller of the restart, and no non-local exit is performed (i.e. the restart declines).

See Also:

"Lisp top-level", "The Read-Eval-Print Loop and the Debugger", clLispTopLevel, exit (function).

Notes:

The clLispTopLevel establishes an exit restart which, when invoked interactively, prompts on *query-io* for confirmation that the user really wants to exit Lisp. The exitp argument to the restart will be true if the user agrees, and false otherwise. The status argument is NIL in either case.

Regardless of how it is invoked, the exit restart established by clLispTopLevel immediately returns status to the caller of clLispTopLevel if exitp is true, and returns NIL as the value of invoke-restart or invoke-restart-interactively if exitp is false.
